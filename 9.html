9. static

1. Co to jest static?

static to słowo kluczowe o różnych znaczeniach w zależności od kontekstu:
-
Zmienne statyczne (w funkcjach i klasach)
-
Funkcje statyczne (w klasach)
-
Składowe statyczne klas
-
Zmienne globalne w przestrzeni pliku

2. Główne zastosowania static

2.1. Zmienne statyczne w funkcjach

Zachowanie:
- Inicjalizowane
tylko raz (przy pierwszym wywołaniu funkcji)
- Zachowują wartość
pomiędzy wywołaniami funkcji
-
Czas życia: od pierwszego użycia do końca programu

Składnia:

void funkcja() {
    static int licznik = 0; // Inicjalizacja tylko raz
    licznik++;
    std::cout << licznik << "\n";
}

Przykład użycia:

void log(const std::string& msg) {
    static int call_count = 0; // Licznik wywołań
    std::cout << "Log #" << ++call_count << ": " << msg << "\n";
}

2.2. Składowe statyczne klas

Właściwości:
- Współdzielone przez
wszystkie instancje klasy
- Istnieją
nawet bez tworzenia obiektów klasy
- Muszą być
definiowane poza klasą

Składnia:

class Klasa {
public:
    static int wspolna_wartosc; // Deklaracja
};

int Klasa::wspolna_wartosc = 0; // Definicja

Przykład użycia:

class Licznik {
public:
    static int count; // Licznik instancji
    
    Licznik() { count++; }
    ~Licznik() { count--; }
};

int Licznik::count = 0; // Inicjalizacja

2.3. Funkcje statyczne w klasach

Charakterystyka:
- Nie mają dostępu do
niestatycznych składowych klasy
- Mogą być wywoływane
bez instancji klasy
- Mają dostęp do
prywatnych składowych statycznych

Składnia:

class Matematyka {
public:
    static double kwadrat(double x) {
        return x * x;
    }
};

Przykład użycia:

double wynik = Matematyka::kwadrat(5.0); // Wywołanie bez obiektu

2.4. Zmienne globalne statyczne

Zachowanie:
- Widoczne
tylko w bieżącym pliku (unit)
- Unikają
konfliktów nazw między plikami

Składnia:

static int globalna = 42; // Widoczna tylko w tym pliku

3. Static w kontekście klas vs funkcji

WłaściwośćStatic w funkcjiStatic w klasie
Czas życiaCały programCały program
WidocznośćLokalna w funkcjiWspółdzielona w klasie
InicjalizacjaPrzy pierwszym użyciuPoza klasą
Dostęp do obiektuBrakTylko do static

4. Zaawansowane zastosowania

4.1. Singleton (wzorzec projektowy)

class Singleton {
public:
    static Singleton& instance() {
        static Singleton inst; // Tylko jedna instancja
        return inst;
    }
    
    // Usunięcie możliwości kopiowania
    Singleton(const Singleton&) = delete;
    Singleton& operator=(const Singleton&) = delete;

private:
    Singleton() {} // Prywatny konstruktor
};

4.2. Metaprogramowanie szablonowe

template<typename T>
class Wrapper {
public:
    static constexpr int size = sizeof(T); // Statyczna wartość
};

5. Najczęstsze błędy

  1. Brak definicji składowych statycznych:
    class Klasa { static int x; };
// Brak: int Klasa::x = 0;
  1. Próba użycia niestatycznych składowych w metodach statycznych
  1. Nadmierne użycie zmiennych statycznych (utrudnia testowanie)

6. Podsumowanie - kiedy używać static

✔ Liczniki i współdzielone zasoby

✔ Funkcje narzędziowe niezależne od stanu obiektu

✔ Dane konfiguracyjne wspólne dla wszystkich instancji

✔ Optymalizacja (unikanie wielokrotnej inicjalizacji)

❌ Unikaj gdy potrzebujesz wielu niezależnych stanów

❌ Uważaj na wyścigi w wielowątkowości (static nie jest thread-safe)

static to potężne narzędzie, ale wymaga ostrożnego stosowania! 🛠️
