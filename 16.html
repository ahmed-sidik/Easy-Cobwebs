16. Wielowatkowosc

Zakres zagadnień:
- mutex, lock
- async
- atomic
- thread
- chrono (pomiar i redukcja narzutu czasowego)

1. Co to jest wielowątkowość w C++?

Mechanizm pozwalający na równoległe wykonywanie wielu sekwencji instrukcji (wątków) w obrębie jednego procesu. W C++ obsługiwana przez bibliotekę <thread>, <mutex>, <atomic>, <future> i <chrono>.

2. Po co wykorzystujemy wielowątkowość?

  • Wykorzystanie wielordzeniowych procesorów - równoległe przetwarzanie
  • Responsywność UI - główny wątek nie blokuje interfejsu
  • Operacje I/O - oczekiwanie na dane nie blokuje całego programu
  • Przyspieszenie obliczeń - podział zadań na części

Thread (std::thread)

1. Co to jest std::thread?

Klasa reprezentująca pojedynczy wątek wykonujący kod współbieżnie.

2. Po co wykorzystujemy?

  • Uruchamianie równoległych zadań
  • Asynchroniczne przetwarzanie
  • Wykonywanie długotrwałych operacji w tle

3. Składnia:

#include <thread>

// Tworzenie wątku
std::thread t1(funkcja); // funkcja bez argumentów
std::thread t2(funkcja, arg1, arg2); // funkcja z argumentami
std::thread t3([](){ /* lambda */ }); // wyrażenie lambda

// Oczekiwanie na zakończenie
t1.join(); 

// Odłączenie wątku (nie czekamy na zakończenie)
t2.detach();

// Przenoszenie wątków
std::thread t4 = std::move(t3);

4. Przykłady użycia:

void print_hello(int n) {
    std::cout << "Hello " << n << "\n";
}

int main() {
    std::thread t1(print_hello, 42);
    std::thread t2([](){ std::cout << "Lambda\n"; });
    
    t1.join();
    t2.join();
}

Mutex i Lock (std::mutex, std::lock_guard)

1. Co to jest?

  • mutex - mechanizm synchronizacji blokujący dostęp do współdzielonych danych
  • lock_guard - RAII wrapper zapewniający bezpieczne zarządzanie mutexem

2. Po co wykorzystujemy?

  • Zapobieganie wyścigom (race conditions)
  • Bezpieczny dostęp do współdzielonych zasobów
  • Synchronizacja między wątkami

3. Składnia:

#include <mutex>

std::mutex mtx;

// Blokowanie/odblokowywanie ręczne
mtx.lock();
// sekcja krytyczna
mtx.unlock();

// Automatyczne zarządzanie (zalecane)
{
    std::lock_guard<std::mutex> lock(mtx);
    // sekcja krytyczna
} // mutex zwolniony automatycznie

// Dla wielu mutexów naraz
std::scoped_lock lock(mtx1, mtx2);

4. Przykłady użycia:

std::mutex mtx;
int shared_data = 0;

void increment() {
    std::lock_guard<std::mutex> lock(mtx);
    ++shared_data;
}

int main() {
    std::thread t1(increment);
    std::thread t2(increment);
    
    t1.join();
    t2.join();
    
    std::cout << shared_data; // Zawsze 2
}

Async (std::async)

1. Co to jest?

Abstrakcja wysokiego poziomu do asynchronicznego wykonywania zadań.

2. Po co wykorzystujemy?

  • Uproszczenie kodu asynchronicznego
  • Automatyczne zarządzanie wątkami
  • Pobieranie wyników z przyszłości (future)

3. Składnia:

#include <future>

// Uruchomienie asynchroniczne
auto future = std::async(std::launch::async, funkcja, args...);

// Uruchomienie leniwe (dopiero przy get())
auto future = std::async(std::launch::deferred, funkcja, args...);

// Pobranie wyniku
auto result = future.get(); // blokuje do zakończenia

4. Przykłady użycia:

int compute(int x) { return x * x; }

int main() {
    auto fut = std::async(std::launch::async, compute, 5);
    // ... inne operacje ...
    std::cout << "Result: " << fut.get(); // 25
}

Atomic (std::atomic)

1. Co to jest?

Typ zapewniający bezpieczne operacje na danych współdzielonych bez użycia muteksów.

2. Po co wykorzystujemy?

  • Operacje atomowe (niepodzielne)
  • Lepsza wydajność niż mutex dla prostych operacji
  • Gwarancja braku wyścigów

3. Składnia:

#include <atomic>

std::atomic<int> counter(0);

// Operacje:
counter.store(42); // atomowy zapis
int val = counter.load(); // atomowe odczytanie
val = counter++; // atomowy increment

4. Przykłady użycia:

std::atomic<int> counter(0);

void increment() {
    for(int i = 0; i < 1000; ++i)
        ++counter;
}

int main() {
    std::thread t1(increment);
    std::thread t2(increment);
    
    t1.join();
    t2.join();
    
    std::cout << counter; // Zawsze 2000
}

Chrono (pomiar czasu)

1. Co to jest?

Biblioteka do precyzyjnego pomiaru czasu i zarządzania czasem.

2. Po co wykorzystujemy?

  • Pomiar wydajności kodu
  • Ograniczanie narzutu czasowego
  • Precyzyjne opóźnienia

3. Składnia:

#include <chrono>
using namespace std::chrono;

// Pomiar czasu
auto start = high_resolution_clock::now();
// kod do zmierzenia
auto end = high_resolution_clock::now();
auto duration = duration_cast<microseconds>(end - start);

// Opóźnienie
std::this_thread::sleep_for(100ms); // C++14
std::this_thread::sleep_for(std::chrono::milliseconds(100));

4. Przykłady użycia:

auto measure() {
    auto start = std::chrono::high_resolution_clock::now();
    
    // Operacja do zmierzenia
    std::this_thread::sleep_for(100ms);
    
    auto end = std::chrono::high_resolution_clock::now();
    return std::chrono::duration_cast<std::chrono::milliseconds>(end - start);
}

int main() {
    auto time = measure();
    std::cout << "Operation took " << time.count() << "ms\n";
}

Redukcja narzutu czasowego

Techniki optymalizacji:

  1. Minimalizacja blokujących operacji - używaj atomic zamiast mutex gdzie możliwe
  1. Thread pooling - unikaj częstego tworzenia/niszczenia wątków
  1. Lock-free programming - algorytmy bez blokowania
  1. Batch processing - grupowanie małych zadań
  1. Affinity - przypinanie wątków do rdzeni (np. przez pthread_setaffinity_np)

Przykład thread pool:

std::vector<std::thread> pool;
for(int i = 0; i < std::thread::hardware_concurrency(); ++i) {
    pool.emplace_back([](){
        while(true) {
            Task task = get_next_task();
            task.execute();
        }
    });
}

Te mechanizmy tworzą kompleksowy zestaw narzędzi do efektywnego programowania wielowątkowego w C++. Pamiętaj, że niewłaściwe użycie może prowadzić do deadlocków, wyścigów i problemów z wydajnością.
