8. enable_if

1. Co to jest std::enable_if?

std::enable_if to metaprogramowanie szablonowe w C++, które pozwala na warunkowe włączanie lub wyłączanie przeciążeń funkcji i specjalizacji szablonów w zależności od spełnienia określonych warunków. Jest to technika oparta na SFINAE (Substitution Failure Is Not An Error), która pozwala kompilatorowi odrzucić nieodpowiednie szablony bez powodowania błędów.

Główne zastosowania:

  • Kontrola przeciążeń funkcji szablonowych.
  • Ograniczanie typów w szablonach.
  • Implementacja traitów typów.

2. Po co wykorzystujemy std::enable_if?

  • Selekcja przeciążeń: Wybór odpowiedniej wersji funkcji w zależności od typu.
  • Ograniczanie typów: Wymuszanie, aby szablon działał tylko dla określonych typów (np. tylko liczby całkowite).
  • Unikanie niejednoznaczności: Zapobieganie konfliktom między podobnymi szablonami.
  • Zastępowane przez Concepts (C++20): W nowoczesnym C++ lepiej używać Concepts, ale enable_if jest nadal używany w starszym kodzie.

3. Składnia

Podstawowa deklaracja std::enable_if

template <bool Condition, typename T = void>  
struct enable_if;  

// Specjalizacja dla Condition == true  
template <typename T>  
struct enable_if<true, T> {  
    using type = T;  
};  

// Dla Condition == false - brak definicji `type` → SFINAE

Użycie enable_if w funkcjach

// Przeciążenie dostępne tylko dla typów całkowitych
template <typename T>
typename std::enable_if<std::is_integral<T>::value, void>::type
foo(T x) {
    std::cout << "Dla typów całkowitych: " << x << "\n";
}

// Przeciążenie dostępne tylko dla typów zmiennoprzecinkowych
template <typename T>
typename std::enable_if<std::is_floating_point<T>::value, void>::type
foo(T x) {
    std::cout << "Dla typów zmiennoprzecinkowych: " << x << "\n";
}

Skrócona wersja z std::enable_if_t (C++14)

template <typename T>
std::enable_if_t<std::is_integral_v<T>, void>  // std::is_integral_v<T> == std::is_integral<T>::value
bar(T x) {
    std::cout << "Tylko dla integral: " << x << "\n";
}

enable_if w parametrach szablonu

template <typename T, typename = std::enable_if_t<std::is_integral_v<T>>>
void baz(T x) {
    std::cout << "Tylko dla integral (parametr szablonu): " << x << "\n";
}

enable_if w zwracanym typie

template <typename T>
auto qux(T x) -> std::enable_if_t<std::is_floating_point_v<T>, void> {
    std::cout << "Tylko dla floating point (zwracany typ): " << x << "\n";
}

4. Przykłady użycia

Przykład 1: Funkcja tylko dla typów całkowitych

#include <iostream>
#include <type_traits>

template <typename T>
typename std::enable_if<std::is_integral<T>::value, void>::type
print_if_integral(T x) {
    std::cout << "To jest liczba całkowita: " << x << "\n";
}

int main() {
    print_if_integral(42);    // OK
    // print_if_integral(3.14); // Błąd kompilacji (nie jest integral)
}

Przykład 2: Przeciążenia dla różnych kategorii typów

#include <iostream>
#include <type_traits>

// Dla typów całkowitych
template <typename T>
std::enable_if_t<std::is_integral_v<T>, void> process(T x) {
    std::cout << "Przetwarzam liczbę całkowitą: " << x << "\n";
}

// Dla typów zmiennoprzecinkowych
template <typename T>
std::enable_if_t<std::is_floating_point_v<T>, void> process(T x) {
    std::cout << "Przetwarzam liczbę zmiennoprzecinkową: " << x << "\n";
}

int main() {
    process(10);      // Wywoła wersję dla integral
    process(3.14);    // Wywoła wersję dla floating point
    // process("text"); // Błąd: brak dopasowania
}

Przykład 3: SFINAE w klasach (specjalizacja szablonu)

#include <iostream>
#include <type_traits>

// Bazowy szablon (nieaktywny)
template <typename T, typename = void>
struct Processor {
    static void process() {
        std::cout << "Domyślna implementacja\n";
    }
};

// Specjalizacja tylko dla typów iterowalnych (np. vector, list)
template <typename T>
struct Processor<T, std::void_t<decltype(std::declval<T>().begin())>> {
    static void process() {
        std::cout << "Typ jest iterowalny!\n";
    }
};

int main() {
    Processor<int>::process();           // Domyślna implementacja
    Processor<std::vector<int>>::process(); // "Typ jest iterowalny!"
}

Przykład 4: enable_if w konstruktorach

#include <iostream>
#include <type_traits>

class Wrapper {
public:
    // Konstruktor tylko dla typów całkowitych
    template <typename T>
    Wrapper(T x, std::enable_if_t<std::is_integral_v<T>, int> = 0) {
        std::cout << "Konstruktor dla integral: " << x << "\n";
    }

    // Konstruktor tylko dla typów wskaźnikowych
    template <typename T>
    Wrapper(T* ptr, std::enable_if_t<std::is_pointer_v<T*>, int> = 0) {
        std::cout << "Konstruktor dla wskaźników: " << ptr << "\n";
    }
};

int main() {
    Wrapper w1(42);        // OK (integral)
    int x = 10;
    Wrapper w2(&x);        // OK (wskaźnik)
    // Wrapper w3(3.14);    // Błąd: nie jest integral ani wskaźnikiem
}

5. Powiązane zagadnienia

SFINAE (Substitution Failure Is Not An Error)

  • Jeśli podstawienie szablonu nie powiedzie się, kompilator nie zgłasza błędu, tylko pomija tę specjalizację.
  • Umożliwia przeciążanie funkcji szablonowych na podstawie cech typów.

std::void_t (C++17)

  • Uproszczona metaprogramowania SFINAE:
template <typename...>
using void_t = void;
  • Używany do sprawdzania, czy wyrażenie jest poprawne.

if constexpr (C++17)

  • Alternatywa dla enable_if w niektórych przypadkach:
template <typename T>
void process(T x) {
    if constexpr (std::is_integral_v<T>) {
        std::cout << "Integral: " << x << "\n";
    } else if constexpr (std::is_floating_point_v<T>) {
        std::cout << "Floating: " << x << "\n";
    }
}

Concepts (C++20) – nowoczesna alternatywa

template <std::integral T>
void better_print(T x) { /* ... */ }  // Czytelniejsze niż enable_if!
