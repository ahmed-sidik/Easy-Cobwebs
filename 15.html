15. Debugowanie

1. Narzędzia do debugowania

1.1 Debugery

NarzędzieSystemCharakterystyka

1.2 Kompilacja do debugowania

g++ -g -O0 -Wall -Wextra program.cpp -o program
  • g: Generuje informacje debugowe
  • O0: Wyłącza optymalizacje (dla lepszego debugowania)
  • Wall -Wextra: Włącza ostrzeżenia

3. Zaawansowane techniki debugowania

3.1 Warunkowe breakpointy

break plik.cpp:45 if x == 42  # GDBb plik.cpp:45 -c "x == 42"    # LLDB

3.2 Watchpointy (śledzenie zmiennych)

watch x                       # GDB/LLDBwatch *(int*)0x7ffeedf123     # Śledzenie adresu pamięci

3.3 Debugowanie wielowątkowe

info threads                  # GDBthread info                   # LLDBthread apply all bt           # Backtrace wszystkich wątków

3.4 Debugowanie z core dump

gdb ./program core            # Analiza zrzutu pamięci

5. Techniki debugowania kodu

5.1 Logowanie debugowe

#ifdef DEBUG
#define DEBUG_LOG(x) std::cerr << x << '\n'
#else
#define DEBUG_LOG(x)
#endif

void funkcja() {
    DEBUG_LOG("Wejście do funkcji");
    // ...
    DEBUG_LOG("Wyjście z funkcji");
}

5.2 Asercje

#include <cassert>

void process(int* data) {
    assert(data != nullptr && "Data cannot be null");
    // ...
}

5.3 Debugowanie RAII

class DebugTimer {
public:
    DebugTimer(const std::string& msg) : msg(msg), start(std::chrono::high_resolution_clock::now()) {}
    ~DebugTimer() {
        auto end = std::chrono::high_resolution_clock::now();
        std::cout << msg << ": " << std::chrono::duration_cast<std::chrono::milliseconds>(end-start).count() << "ms\n";
    }
private:
    std::string msg;
    std::chrono::time_point<std::chrono::high_resolution_clock> start;
};

void funkcja() {
    DebugTimer timer("Czas wykonania funkcji");
    // ...
}

6. Narzędzia diagnostyczne

6.1 Sanitizers

SanitizerWykrywaKompilacja
AddressSanitizerBłędy pamięci-fsanitize=address
UndefinedBehaviorNiezdefiniowane zachowania-fsanitize=undefined
ThreadSanitizerWyścigi w wątkach-fsanitize=thread

Przykład użycia:

g++ -fsanitize=address -g program.cpp -o program

6.2 Valgrind

valgrind --leak-check=full ./program
  • Wykrywa wycieki pamięci
  • Analiza użycia pamięci
  • Profilowanie cache

6.3 Static Analyzers

clang-tidy --checks=* program.cpp
cppcheck --enable=all program.cpp

7. Debugowanie zaawansowanych problemów

7.1 Wyścigi (Race Conditions)

  1. Kompiluj z fsanitize=thread
  1. Użyj ThreadSanitizer
  1. Debuguj z watchpointami na współdzielonych danych

7.2 Wycieki pamięci

valgrind --leak-check=full --show-leak-kinds=all ./program

7.3 Deadlocki

  1. Użyj bt full w debuggerze
  1. Analizuj backtrace wszystkich wątków
  1. Szukaj wzorców blokowania muteksów

8. Debugowanie w praktyce - case study

8.1 Segmentacja fault (SIGSEGV)

  1. Uruchom z debuggerem
  1. Po crashu wykonaj bt (backtrace)
  1. Sprawdź wartości wskaźników przed crash-em
  1. Użyj AddressSanitizer

8.2 Nieskończona pętla

  1. Wstrzymaj program (Ctrl+C w debuggerze)
  1. Sprawdź backtrace
  1. Zbadaj warunki pętli
  1. Użyj watchpointów na zmiennych sterujących

8.3 Nieprawidłowe wyniki

  1. Ustaw breakpointy w kluczowych miejscach
  1. Śledź wartości zmiennych
  1. Sprawdź przypadki brzegowe
  1. Użyj asercji i testów jednostkowych

9. Najlepsze praktyki debugowania

  1. Reprodukowalność: Zawsze miej sposób na reprodukcję błędu
  1. Divide and Conquer: Zawężaj obszar poszukiwań
  1. Binary Search: W kodzie użyj metody połowienia
  1. Rubber Duck Debugging: Opisz problem komuś/czemuś (nawet kaczce)
  1. Version Bisect: Użyj git bisect do znalezienia wprowadzającej błąd zmiany
  1. Minimal Working Example: Stwórz minimalny kod reprodukujący problem
  1. Zapisuj swoje kroki: Dokumentuj co już sprawdziłeś

Debugowanie to nieodłączna część procesu rozwoju oprogramowania. Im lepiej opanujesz narzędzia i techniki debugowania, tym szybciej będziesz identyfikować i naprawiać problemy w kodzie. Pamiętaj, że dobry programista nie to taki, który nie popełnia błędów, ale taki, który potrafi je skutecznie znajdować i naprawiać.
