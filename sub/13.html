13. Wyjatki

Zakres zagadnień:
- exception
- try-catch

1. Co to są wyjątki w C++?

Mechanizm obsługi błędów pozwalający na przerwanie normalnego przepływu programu i przekazanie kontroli do specjalnego bloku obsługi błędów. Składa się z:
-
throw - zgłoszenie wyjątku
-
try - blok chroniony
-
catch - blok obsługi wyjątku

2. Po co wykorzystujemy wyjątki?

  • Separacja logiki biznesowej od obsługi błędów
  • Centralna obsługa błędów - unikanie powielania kodu obsługi błędów
  • Przekazywanie błędów przez wiele warstw wywołań
  • Bezpieczne zwalnianie zasobów (przez RAII)
  • Lepsza czytelność niż kod oparty na kodach błędów

3. Składnia podstawowa

Zgłaszanie wyjątku:

throw typ_wyjątku(argumenty);
// np.:
throw std::runtime_error("Błąd otwarcia pliku");

Blok try-catch:

try {
    // kod który może zgłosić wyjątek
} 
catch (const typ1& e) {
    // obsługa wyjątku typu typ1
} 
catch (const typ2& e) {
    // obsługa wyjątku typu typ2
} 
catch (...) {
    // obsługa wszystkich innych wyjątków
}

5. Przykłady użycia

Przykład 1: Podstawowa obsługa błędów

double dziel(double a, double b) {
    if (b == 0.0) 
        throw std::invalid_argument("Dzielenie przez zero");
    return a / b;
}

int main() {
    try {
        double wynik = dziel(10, 0);
    }
    catch (const std::invalid_argument& e) {
        std::cerr << "Błąd: " << e.what() << "\n";
    }
}

Przykład 2: Własna klasa wyjątku

class MojaKlasaBledu : public std::exception {
    std::string msg;
public:
    MojaKlasaBledu(const std::string& s) : msg(s) {}
    const char* what() const noexcept override {
        return msg.c_str();
    }
};

void test() {
    throw MojaKlasaBledu("Coś poszło nie tak");
}

Przykład 3: Wielokrotne catch

try {
    // kod który może zgłosić różne wyjątki
} 
catch (const std::out_of_range& e) {
    // obsługa błędów zakresu
} 
catch (const std::logic_error& e) {
    // obsługa innych błędów logicznych
} 
catch (const std::exception& e) {
    // obsługa wszystkich pozostałych wyjątków standardowych
} 
catch (...) {
    // obsługa wszystkich innych wyjątków
}

6. Zaawansowane techniki

Exception safety gwarancje:

  1. No-throw - operacja nigdy nie zgłasza wyjątków (noexcept)
  1. Basic - brak wycieków zasobów, obiekt pozostaje w spójnym stanie
  1. Strong - operacja albo się powiedzie, albo nie zmieni stanu programu
  1. No guarantee - brak gwarancji

Specyfikacja noexcept (C++11):

void funkcja_gwarantowana() noexcept; // nigdy nie throw
void inna_funkcja() noexcept(true); // może być w czasie kompilacji

Przekazywanie wyjątków między wątkami:

std::exception_ptr eptr;

void watek_funkcja() {
    try {
        // kod wątku
    } catch (...) {
        eptr = std::current_exception();
    }
}

int main() {
    std::thread t(wątek_funkcja);
    t.join();
    
    if (eptr) {
        try {
            std::rethrow_exception(eptr);
        } catch (const std::exception& e) {
            std::cout << e.what();
        }
    }
}

7. Najlepsze praktyki

  1. Nie używaj wyjątków do kontroli przepływu - tylko do sytuacji wyjątkowych
  1. Dziedzicz z std::exception dla własnych wyjątków
  1. Przechwytuj wyjątki przez referencję (catch (const T& e))
  1. Zawsze obsługuj wyjątki w wątkach - niezgłoszone wyjątki kończą program
  1. Używaj RAII do zarządzania zasobami (np. smart pointers)
  1. Unikaj throw w destruktorach - może prowadzić do std::terminate
  1. Dokumentuj jakie wyjątki mogą być zgłaszane przez funkcje

8. Exception handling a wydajność

  • Koszt try-catch jest minimalny gdy wyjątek nie jest zgłaszany
  • Zgłoszenie wyjątku jest kosztowne (ok. 10-100x wolniejsze niż kod błędów)
  • Optymalizacja: Używaj wyjątków tylko dla naprawdę wyjątkowych sytuacji
auto start = std::chrono::high_resolution_clock::now();
for (int i = 0; i < 1000000; ++i) {
    try {
        throw 1;
    } catch (int) {}
}
auto duration = std::chrono::duration_cast<std::chrono::milliseconds>(
    std::chrono::high_resolution_clock::now() - start
);
std::cout << "Czas: " << duration.count() << "ms\n";

9. Ciekawostki

  • W C++ nie ma finally - używaj RAII (np. std::lock_guard)
  • Wyjątki mogą być dowolnego typu (int, string, ale nie zalecane)
  • Dynamiczne wyjątki (throw()) są deprecated od C++11
  • W C++17 wprowadzono std::uncaught_exceptions() - liczba aktywnych wyjątków

Te informacje powinny zapewnić solidne podstawy do pracy z wyjątkami w C++. Pamiętaj, że dobrze zaprojektowany system obsługi błędów jest kluczowy dla niezawodności oprogramowania.
