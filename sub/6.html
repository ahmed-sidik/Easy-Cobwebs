1. Polimorfizm

1.1 Co to jest?

Mechanizm pozwalający na różne zachowania tych samych metod w zależności od typu obiektu.

1.2 Rodzaje polimorfizmu:

  • Statyczny (compile-time):
    • Przeciążanie funkcji
    • Przeciążanie operatorów
    • Szablony
  • Dynamiczny (runtime):
    • Metody wirtualne
    • Dziedziczenie

1.3 Przykład polimorfizmu dynamicznego:

class Shape {
public:
    virtual void draw() const = 0; // czysto wirtualna
    virtual ~Shape() {}
};

class Circle : public Shape {
public:
    void draw() const override {
        std::cout << "Drawing Circle\n";
    }
};

class Square : public Shape {
public:
    void draw() const override {
        std::cout << "Drawing Square\n";
    }
};

void render(const std::vector<Shape*>& shapes) {
    for (const auto& shape : shapes) {
        shape->draw(); // dynamiczne wiązanie
    }
}

2. Enkapsulacja

2.1 Co to jest?

Zasada ukrywania danych i udostępniania kontrolowanego dostępu przez publiczne metody.

2.2 Poziomy dostępu:

ModyfikatorDostęp
publicDostęp wszędzie
protectedKlasa i klasy pochodne
privateTylko klasa

2.3 Przykład enkapsulacji:

class BankAccount {
private:
    double balance;
    std::string owner;

public:
    BankAccount(std::string owner) : owner(owner), balance(0) {}
    
    void deposit(double amount) {
        if (amount > 0) balance += amount;
    }
    
    bool withdraw(double amount) {
        if (amount > 0 && balance >= amount) {
            balance -= amount;
            return true;
        }
        return false;
    }
    
    double getBalance() const { return balance; }
};

3. Klasy abstrakcyjne

3.1 Co to jest?

Klasa zawierająca co najmniej jedną metodę czysto wirtualną, której nie można instancjonować.

3.2 Składnia:

class AbstractClass {
public:
    virtual void pureVirtual() = 0; // czysto wirtualna
    virtual ~AbstractClass() {}     // wirtualny destruktor
};

3.3 Przykład użycia:

class Animal {
public:
    virtual void makeSound() = 0;
};

class Dog : public Animal {
public:
    void makeSound() override {
        std::cout << "Woof!\n";
    }
};

// Animal animal; // Błąd - klasa abstrakcyjna
Dog dog;         // OK

4. Metody wirtualne

4.1 Metody wirtualne vs czysto wirtualne:

CechaMetoda wirtualnaCzysto wirtualna
Składniavirtual void func()virtual void func() = 0
ImplementacjaMa domyślną implementacjęBrak implementacji
Wymusza overrideNieTak
Klasa abstrakcyjnaNieTak

4.2 Zasady:

  • Destruktory powinny być wirtualne w klasach bazowych
  • Używaj override dla jasności kodu (C++11)
  • Unikaj virtual w klasach final

4.3 Przykład:

class Base {
public:
    virtual void func() { std::cout << "Base\n"; }
    virtual ~Base() {}
};

class Derived : public Base {
public:
    void func() override { std::cout << "Derived\n"; }
};

5. Funkcje zaprzyjaźnione (friend)

5.1 Co to jest?

Mechanizm pozwalający funkcjom lub klasom zewnętrznym na dostęp do prywatnych i chronionych składowych klasy.

5.2 Składnia:

class MyClass {
private:
    int secret;
    
    friend void friendFunction(MyClass&);
    friend class FriendClass;
};

void friendFunction(MyClass& obj) {
    obj.secret = 42; // Dostęp do prywatnego pola
}

class FriendClass {
public:
    void modify(MyClass& obj) {
        obj.secret = 100; // Dostęp do prywatnego pola
    }
};

5.3 Zastosowania:

  • Operatory wejścia/wyjścia (<<, >>)
  • Funkcje narzędziowe wymagające dostępu do prywatnych danych
  • Testy jednostkowe

5.4 Ostrzeżenia:

  • Narusza enkapsulację
  • Powinien być używany oszczędnie
  • Nie jest dziedziczony

6. Przeciążanie operatorów

6.1 Co to jest?

Definiowanie działania operatorów dla własnych typów.

6.2 Lista przeciążalnych operatorów:

+ - * / % ^ & | ~ ! = < > += -= *= /= %= ^= &= |= << >> >>= <<= == != <= >= && || ++ -- , ->* -> () [] new new[] delete delete[]

6.3 Przykłady:

Operator arytmetyczny:

class Vector {
public:
    Vector operator+(const Vector& other) const {
        return Vector(x + other.x, y + other.y);
    }
private:
    double x, y;
};

Operator strumienia:

friend std::ostream& operator<<(std::ostream& os, const Vector& v) {
    return os << "[" << v.x << ", " << v.y << "]";
}

Operator indeksowania:

class Array {
public:
    int& operator[](size_t index) {
        if (index >= size) throw std::out_of_range("Index out of range");
        return data[index];
    }
private:
    int* data;
    size_t size;
};

Operator konwersji:

operator std::string() const {
    return "Vector to string";
}

6.4 Zasady przeciążania:

  • Zachowaj naturalne znaczenie operatora
  • Nie przeciążaj operatorów w nieintuicyjny sposób
  • Zwracaj referencje dla =, +=, itp.
  • Deklaruj jako const gdy nie modyfikują obiektu

7. Przeciążanie metod

7.1 Co to jest?

Definiowanie wielu funkcji o tej samej nazwie ale różnych parametrach.

7.2 Przykład:

class Printer {
public:
    void print(int value) {
        std::cout << "Integer: " << value << "\n";
    }
    
    void print(double value) {
        std::cout << "Double: " << value << "\n";
    }
    
    void print(const std::string& text) {
        std::cout << "String: " << text << "\n";
    }
};

7.3 Zasady:

  • Różne sygnatury parametrów (typ, liczba, kolejność)
  • Nie można przeciążać tylko po typie zwracanym
  • Można łączyć z metodami wirtualnymi

8. Kompletny przykład OOP

#include <iostream>
#include <string>
#include <vector>

// Klasa abstrakcyjna
class Animal {
public:
    Animal(const std::string& name) : name(name) {}
    virtual ~Animal() {}
    
    // Czysto wirtualna
    virtual std::string sound() const = 0;
    
    // Wirtualna z implementacją
    virtual void info() const {
        std::cout << name << " says " << sound() << "\n";
    }
    
    // Przeciążona metoda
    void feed() { std::cout << name << " is fed\n"; }
    void feed(const std::string& food) {
        std::cout << name << " is fed with " << food << "\n";
    }
    
protected:
    std::string name;
};

// Dziedziczenie
class Dog : public Animal {
public:
    Dog(const std::string& name) : Animal(name) {}
    
    std::string sound() const override {
        return "Woof!";
    }
    
    // Przeciążenie operatora
    friend std::ostream& operator<<(std::ostream& os, const Dog& dog) {
        return os << "Dog: " << dog.name;
    }
};

class Cat : public Animal {
public:
    Cat(const std::string& name) : Animal(name) {}
    
    std::string sound() const override {
        return "Meow!";
    }
    
    void info() const override {
        std::cout << "Special cat info: ";
        Animal::info(); // Wywołanie metody bazowej
    }
};

int main() {
    Dog dog("Burek");
    Cat cat("Mruczek");
    
    Animal* animals[] = {&dog, &cat};
    
    for (auto animal : animals) {
        animal->info(); // Polimorfizm
    }
    
    dog.feed();
    dog.feed("bone");
    
    std::cout << dog << "\n"; // Operator <<
}

9. Najlepsze praktyki OOP w C++

  1. Zasada pojedynczej odpowiedzialności - klasa powinna mieć tylko jeden powód do zmiany
  1. Prawo Demeter - minimalizuj zależności między klasami
  1. LSP (Liskov Substitution Principle) - klasy pochodne powinny być zastępowalne przez klasy bazowe
  1. Preferuj kompozycję nad dziedziczeniem
  1. Zasada otwarte-zamknięte - klasy powinny być otwarte na rozszerzenie, ale zamknięte na modyfikację
  1. Używaj final gdy klasa/metoda nie powinna być dziedziczona/override’owana
  1. Unikaj protected danych - lepiej użyć prywatnych z protected metodami dostępowymi
  1. Wirtualne destruktory w klasach bazowych
  1. Unikaj przyjaźni między klasami jeśli to możliwe
  1. Przeciążaj operatory intuicyjnie - nie zmieniaj ich naturalnego znaczenia
