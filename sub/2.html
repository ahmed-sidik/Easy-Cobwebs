2. Move semantics

1. Semantyka przenoszenia (Move Semantics)

Podstawowe pojęcia

  • std::move - nie przenosi sam, tylko rzutuje na rvalue referencję
  • Konstruktor przenoszący i operator przypisania przenoszącego
  • Zastosowanie: optymalizacja operacji na dużych obiektach (np. kontenery STL)

Przykład konstruktora przenoszącego:

class Buffer {
    char* data;
    size_t size;
public:
    // Konstruktor przenoszący
    Buffer(Buffer&& other) noexcept 
        : data(other.data), size(other.size) {
        other.data = nullptr;  // ważne: unieważniamy źródło!
        other.size = 0;
    }
    
    // Operator przypisania przenoszącego
    Buffer& operator=(Buffer&& other) noexcept {
        if (this != &other) {
            delete[] data;      // zwalniamy obecne zasoby
            data = other.data;  // przejmujemy dane
            size = other.size;
            other.data = nullptr;
            other.size = 0;
        }
        return *this;
    }
};

2. Operacje swap, copy i move

std::swap

  • Wydajna zamiana zasobów między obiektami
  • Domyślna implementacja używa kopiowania (można specjalizować)

Optymalna implementacja swap:

class Resource {
    friend void swap(Resource& a, Resource& b) noexcept {
        using std::swap;
        swap(a.data, b.data);  // ADL (Argument Dependent Lookup)
        swap(a.size, b.size);
    }
};

std::copy vs std::move

OperacjaEfektWydajnośćUżycie
std::copyGłęboka kopiaNiskaDla małych obiektów
std::movePłytka kopiaWysokaDla dużych obiektów

Przykład:

std::vector<std::string> v1, v2;
// Kopiowanie - O(n)
std::copy(v1.begin(), v1.end(), std::back_inserter(v2));
// Przenoszenie - O(1) per element
std::move(v1.begin(), v1.end(), std::back_inserter(v2));

3. Zarządzanie pamięcią - zaawansowane techniki

Alokatory (Allocators)

  • Kontrola nad alokacją pamięci dla kontenerów
  • Przykład: pool allocator, stack allocator

Niestandardowy alokator:

template <class T>
struct MyAllocator {
    using value_type = T;
    T* allocate(size_t n) {
        return static_cast<T*>(::operator new(n * sizeof(T)));
    }
    void deallocate(T* p, size_t n) {
        ::operator delete(p);
    }
};

std::vector<int, MyAllocator<int>> v;

Obsługa wyjątków przy alokacji

  • std::nothrow - wersja new nie rzucająca wyjątków
  • Obsługa błędów alokacji
int* p = new(std::nothrow) int[1'000'000];
if (!p) {
    // obsługa błędu alokacji
}

4. Optymalizacje wydajnościowe

Small Buffer Optimization (SBO)

  • Przechowywanie małych obiektów na stosie
  • Implementacja w std::string (w wielu implementacjach)

Copy-On-Write (COW)

  • Technika opóźnionego kopiowania
  • Historycznie w std::string (obecnie rzadko stosowane)

5. Inne ważne mechanizmy

std::exchange (C++14)

  • Atomowa zamiana wartości
void clear(Buffer& buf) {
    auto old = std::exchange(buf.data, nullptr);
    delete[] old;
}

std::forward (Perfect Forwarding)

  • Zachowuje kategorię wartości (lvalue/rvalue)
  • Kluczowe w szablonach
template <typename T>
void wrapper(T&& arg) {
    process(std::forward<T>(arg));  // perfect forwarding
}
