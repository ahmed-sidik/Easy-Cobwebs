5. Concepty

1. Co to są Concepts?

Concepts to mechanizm wprowadzony w C++20, który pozwala na określenie wymagań dla szablonów (template’ów). Służą do sprawdzania, czy dany typ lub zestaw typów spełnia określone warunki w czasie kompilacji.

Mówiąc prościej: Concepts definiują, jakie właściwości musi mieć typ, aby można go było użyć w szablonie. Dzięki temu można uniknąć nieczytelnych błędów kompilacji związanych z nieodpowiednimi typami.

2. Po co wykorzystujemy Concepts?

  • Lepsza czytelność błędów: Zamiast długich, niejasnych komunikatów o błędach w szablonach, Concepts precyzyjnie wskazują, jakie wymagania nie zostały spełnione.
  • Wymuszanie ograniczeń: Można jawnie określić, jakie operacje muszą być dostępne dla danego typu.
  • Uproszczenie przeciążania funkcji szablonowych: Można tworzyć różne wersje funkcji dla różnych Concepts.
  • Zastępuje std::enable_if i SFINAE: Concepts są bardziej czytelne i łatwiejsze w użyciu niż metaprogramowanie z SFINAE.

3. Składnia

Deklaracja Conceptu

template <typename T>
concept NazwaConceptu = requires(T a, T b) {
    { a + b } -> std::convertible_to<T>;  // Wymaganie: operator + musi istnieć i zwracać coś konwertowalnego na T
    a == b;                               // Wymaganie: musi istnieć operator ==
};

Użycie Conceptu w funkcji szablonowej

template <NazwaConceptu T>
void funkcja(T arg) { /* ... */ }

Skrócona składnia (C++20)

void funkcja(NazwaConceptu auto arg) { /* ... */ }

Łączenie Concepts

template <typename T>
requires NazwaConceptu1<T> && NazwaConceptu2<T>
void funkcja(T arg) { /* ... */ }

Standardowe Concepts (z <concepts> i <iterator>)

  • std::integral<T> – typ musi być całkowity (np. int, long).
  • std::floating_point<T> – typ musi być zmiennoprzecinkowy (np. float, double).
  • std::equality_comparable<T> – typ musi mieć operator==.
  • std::invocable<F, Args...> – obiekt F musi dać się wywołać z argumentami Args....

4. Przykłady użycia

Przykład 1: Prosty Concept sprawdzający możliwość dodawania

#include <concepts>
#include <iostream>

template <typename T>
concept Addable = requires(T a, T b) {
    { a + b } -> std::same_as<T>;  // Operator + musi zwracać T
};

template <Addable T>
T add(T a, T b) {
    return a + b;
}

int main() {
    std::cout << add(5, 3) << "\n";       // OK: int jest Addable
    // std::cout << add("a", "b");        // Błąd: const char* nie jest Addable
}

Przykład 2: Concept dla typów posortowywalnych

#include <algorithm>
#include <vector>

template <typename Container>
concept Sortable = requires(Container c) {
    { std::sort(c.begin(), c.end()) };  // Wymagamy, aby dało się sortować
};

void sortContainer(Sortable auto& container) {
    std::sort(container.begin(), container.end());
}

int main() {
    std::vector<int> vec = {3, 1, 4, 1, 5};
    sortContainer(vec);  // OK: vector jest Sortable

    // int x = 5;
    // sortContainer(x);  // Błąd: int nie jest kontenerem
}

Przykład 3: Własny Concept dla typów z metodą print()

#include <iostream>

template <typename T>
concept Printable = requires(T obj) {
    { obj.print() } -> std::same_as<void>;  // Wymagamy metody print()
};

struct Person {
    void print() const { std::cout << "Person\n"; }
};

struct Car {};

void callPrint(Printable auto const& obj) {
    obj.print();
}

int main() {
    Person p;
    callPrint(p);  // OK: Person ma print()

    // Car c;
    // callPrint(c);  // Błąd: Car nie ma print()
}

Przykład 4: Użycie standardowego std::integral

#include <concepts>
#include <iostream>

void printNumber(std::integral auto num) {
    std::cout << "Liczba całkowita: " << num << "\n";
}

int main() {
    printNumber(42);     // OK
    // printNumber(3.14); // Błąd: double nie jest integral
}

Podsumowanie

Concepts w C++20 to potężne narzędzie, które pozwala na:

✔ Jawne określanie wymagań dla szablonów

✔ Lepsze komunikaty o błędach

✔ Zastępowanie skomplikowanych technik SFINAE

✔ Łatwiejsze przeciążanie funkcji szablonowych

Warto ich używać, aby pisać bardziej bezpieczny i czytelny kod generyczny. 🚀
