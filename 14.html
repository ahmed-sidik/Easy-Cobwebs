1. Testy jednostkowe (Unit Tests)

1.1 Co to są testy jednostkowe?

Testy weryfikujące pojedyncze jednostki kodu (funkcje, klasy) w izolacji od reszty systemu.

1.2 Po co stosujemy?

  • Wczesne wykrywanie błędów
  • Dokumentacja zachowania kodu
  • Ułatwienie refaktoryzacji
  • Wsparcie dla TDD (Test-Driven Development)

1.3 Popularne frameworki:

  • Google Test (gtest)
  • Catch2
  • Boost.Test
  • doctest

1.4 Przykład z Google Test:

#include <gtest/gtest.h>

int add(int a, int b) { return a + b; }

TEST(MathTest, Addition) {
    EXPECT_EQ(add(2, 3), 5);
    EXPECT_NE(add(0, 0), 1);
    ASSERT_TRUE(add(-1, 1) == 0);
}

int main(int argc, char **argv) {
    ::testing::InitGoogleTest(&argc, argv);
    return RUN_ALL_TESTS();
}

1.5 Typowe asercje:

AsercjaZnaczenie
ASSERT_TRUE(x)x == true (abort jeśli fail)
EXPECT_EQ(a,b)a == b
EXPECT_NEAR(a,b,eps)
EXPECT_THROW(stmt, exc)stmt rzuca exc

2. Testy integracyjne

2.1 Co to są testy integracyjne?

Testy weryfikujące współdziałanie wielu komponentów systemu.

2.2 Po co stosujemy?

  • Wykrywanie błędów w interfejsach między modułami
  • Weryfikacja przepływu danych
  • Testowanie integracji z zewnętrznymi systemami

2.3 Przykład testu integracyjnego:

TEST(IntegrationTest, DatabaseAndLogic) {
    Database db = connectToTestDB();
    BusinessLogic logic(db);
    
    logic.processData("test_input");
    auto result = db.query("SELECT * FROM results");
    
    EXPECT_EQ(result.size(), 1);
    EXPECT_EQ(result[0].status, "processed");
}

2.4 Strategie:

  • Big Bang - integracja wszystkich modułów naraz
  • Inkrementalna:
    • Od dołu (bottom-up)
    • Od góry (top-down)
    • Sandwich (hybryda)

3. Mockowanie

3.1 Co to jest mock?

Atrapa obiektu symulująca zachowanie rzeczywistego komponentu.

3.2 Po co stosujemy?

  • Izolacja testowanej jednostki
  • Symulowanie trudnych przypadków (błędy, wyjątki)
  • Weryfikacja interakcji
  • Przyspieszenie testów (np. zamiast prawdziwej bazy danych)

3.3 Popularne biblioteki:

  • Google Mock (gmock)
  • FakeIt
  • Hippomocks

3.4 Przykład z Google Mock:

class DatabaseInterface {
public:
    virtual ~DatabaseInterface() {}
    virtual bool save(const std::string &data) = 0;
};

class MockDB : public DatabaseInterface {
public:
    MOCK_METHOD(bool, save, (const std::string &data), (override));
};

TEST(BusinessLogicTest, SaveToDB) {
    MockDB mock_db;
    BusinessLogic logic(mock_db);
    
    // Oczekujemy dokładnie jednego wywołania save z dowolnym argumentem
    EXPECT_CALL(mock_db, save(testing::_))
        .Times(1)
        .WillOnce(testing::Return(true));
    
    logic.processData("test_data");
}

4. Testowanie w C++ - najlepsze praktyki

4.1 Organizacja testów

  • Arrange-Act-Assert (AAA) pattern:
    TEST(TestSuite, TestName) {
      // Arrange - przygotowanie danych
      auto calculator = Calculator();
      
      // Act - wykonanie akcji
      auto result = calculator.add(2, 3);
      
      // Assert - weryfikacja
      EXPECT_EQ(result, 5);
  }

4.2 Zasady F.I.R.S.T:

  • Fast - testy powinny być szybkie
  • Isolated - testy nie powinny zależeć od siebie
  • Repeatable - wyniki powinny być spójne
  • Self-validating - test powinien sam określać sukces/porażkę
  • Timely - pisane przed kodem produkcyjnym (TDD)

4.3 Pokrycie kodu (Code Coverage)

  • Linie kodu (line coverage)
  • Gałęzie (branch coverage)
  • Warunki (condition coverage)

5. Zaawansowane techniki

5.1 Testowanie prywatnych metod

  • Friend class w testowanej klasie
  • FIXTURE_TEST w Google Test
  • Refleksja (w ograniczonym zakresie w C++)

5.2 Parametryzowane testy

class ParamTest : public testing::TestWithParam<int> {};

TEST_P(ParamTest, IsEven) {
    int n = GetParam();
    EXPECT_TRUE(n % 2 == 0);
}

INSTANTIATE_TEST_SUITE_P(EvenNumbers, ParamTest,
    testing::Values(2, 4, 6, 8));

5.3 Testowanie wyjątków

TEST(ExceptionTest, InvalidInput) {
    EXPECT_THROW({
        parseInput("invalid");
    }, std::invalid_argument);
}

5.4 Testy wydajnościowe (benchmarki)

static void BM_StringCopy(benchmark::State& state) {
    std::string x = "hello";
    for (auto _ : state) {
        std::string copy(x);
    }
}
BENCHMARK(BM_StringCopy);
