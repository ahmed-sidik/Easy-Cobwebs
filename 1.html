1. Inteligentne wskazniki i referencje

Zakres zagadnień:
- unique_ptr, weak_ptr, shared_ptr, auto_ptr
- std::move, std::swap, std::copy
- rvalue, lvalue
- dynamiczna alokacja i dealokacja pamięci

1. Inteligentne wskaźniki

std::unique_ptr

Co to jest?

- Wyłączny właściciel zasobu (nie można go kopiować, tylko przenosić).

- Automatycznie zwalnia pamięć przy zniszczeniu.

Po co używać?

- Bezpieczne zarządzanie pamięcią bez wycieków.

- Lepsza wydajność niż shared_ptr (brak narzutu licznika referencji).

Składnia:

#include <memory>  

std::unique_ptr<T> ptr(new T);  // Stara wersja (C++11)  
auto ptr = std::make_unique<T>(); // Nowoczesna wersja (C++14)  

// Przenoszenie własności  
std::unique_ptr<T> ptr2 = std::move(ptr);

Przykład:

auto resource = std::make_unique<int>(42);
std::cout << *resource << "\n";  // Dereferencja

std::shared_ptr

Co to jest?

- Współdzielony wskaźnik z licznikiem referencji.

- Zasób jest zwalniany, gdy ostatni shared_ptr jest niszczony.

Po co używać?

- Gdy wiele obiektów musi współdzielić ten sam zasób.

Składnia:

auto ptr = std::make_shared<T>(); // Zalecane  
std::shared_ptr<T> ptr2(new T);   // Alternatywa  

// Kopiowanie zwiększa licznik referencji  
std::shared_ptr<T> ptr3 = ptr2;

Przykład:

auto shared = std::make_shared<std::string>("Hello");
auto shared2 = shared;  // Licznik referencji = 2

std::weak_ptr

Co to jest?

- Słaba referencja do obiektu zarządzanego przez shared_ptr.

- Nie zwiększa licznika referencji.

Po co używać?

- Zapobieganie cyklom referencji (np. w strukturze grafów).

Składnia:

std::weak_ptr<T> weak = shared_ptr;  

// Konwersja do shared_ptr (jeśli obiekt jeszcze istnieje)  
if (auto locked = weak.lock()) {  
    // Użyj locked jako shared_ptr  
}

Przykład:

auto shared = std::make_shared<int>(42);  
std::weak_ptr<int> weak = shared;  

if (auto temp = weak.lock()) {  
    std::cout << *temp << "\n";  
}

std::auto_ptr (przestarzałe, C++98/03)

  • Zastąpione przez std::unique_ptr.
  • Niebezpieczny (niejawna semantyka przenoszenia).

2. Operacje na wskaźnikach

std::move

  • Przekształca obiekt w rvalue, umożliwiając przenoszenie zasobów.
  • Używane z unique_ptr, std::vector itp.

Przykład:

auto ptr1 = std::make_unique<int>(10);  
auto ptr2 = std::move(ptr1);  // ptr1 jest teraz nullptr

std::swap

  • Zamienia zawartości dwóch obiektów.

Przykład:

std::unique_ptr<int> a(new int(5));  
std::unique_ptr<int> b(new int(10));  
std::swap(a, b);  // a ma 10, b ma 5

std::copy

  • Kopiuje zakres elementów (ale nie działa z unique_ptr!).

3. Rvalue vs Lvalue

LvalueRvalue
Ma adres w pamięci (&x działa).Tymczasowy obiekt (np. wynik funkcji).
Może stać po lewej stronie =.Zwykle po prawej stronie =.
Przykład: int x; x = 5;Przykład: 5, x + y, std::move(x)

std::move konwertuje lvalue na rvalue, umożliwiając przenoszenie zasobów.

4. Dynamiczna alokacja i dealokacja

Tradycyjne zarządzanie (niezalecane)

int* arr = new int[100];  // Alokacja  
delete[] arr;             // Dealokacja (łatwo zapomnieć!)

Z inteligentnymi wskaźnikami

// Alokacja tablicy  
auto arr = std::make_unique<int[]>(100);  

// Nie trzeba ręcznie zwalniać!

Podsumowanie

WskaźnikUżycieZalety
unique_ptrWyłączna własność.Brak narzutu, bezpieczeństwo.
shared_ptrWspółdzielona własność.Proste zarządzanie cyklami.
weak_ptrUnikanie cykli referencji.Bezpieczne “podglądanie”.

Zasady:

1. Zawsze używaj make_unique i make_shared zamiast new.

2. Unikaj auto_ptr – użyj unique_ptr.

3. std::move służy do optymalizacji przenoszenia zasobów.

4. Lepiej używać inteligentnych wskaźników niż new/delete.

🚀 Nowoczesne C++ preferuje RAII i inteligentne wskaźniki!
