10. Wyrazenia lambda

1. Co to są wyrażenia lambda?

Wyrażenia lambda (funkcje anonimowe) to krótkie, lokalne funkcje definiowane inline w kodzie, bez potrzeby deklarowania oddzielnej nazwanej funkcji. Wprowadzone w C++11, stały się fundamentem nowoczesnego C++.

2. Po co wykorzystujemy lambdy?

  • Krótkie, jednorazowe funkcje - gdy nie chcemy tworzyć pełnej funkcji
  • Algorytmy STL - przekazywanie predykatów (np. w std::sort, std::find_if)
  • Asynchroniczne operacje - wygodne przekazywanie callbacków
  • Zamykanie kontekstu - przechwytywanie zmiennych z otaczającego scope’u
  • Czytelność kodu - logika blisko miejsca użycia

3. Pełna składnia lambda

[ przechwytywanie ] ( parametry ) mutable constexpr noexcept -> typ_zwracany {
// ciało lambdy
}

Elementy składowe:

  1. Przechwytywanie ([capture]) - określa dostęp do zmiennych z zewnętrznego scope’u
  1. Parametry - jak w zwykłych funkcjach
  1. Modyfikatory (mutable, constexpr, noexcept)
  1. Typ zwracany (opcjonalny, często dedukowany)
  1. Ciało funkcji

4. Przechwytywanie zmiennych

SkładniaZnaczenie
[]Nie przechwytuje żadnych zmiennych
[x]Przechwytuje x przez wartość (kopia)
[&x]Przechwytuje x przez referencję
[=]Przechwytuje wszystkie zmienne przez wartość
[&]Przechwytuje wszystkie zmienne przez referencję
[x, &y]Mieszane przechwytywanie
[this]Przechwytuje wskaźnik this
[*this]Przechwytuje obiekt przez wartość (C++17)
[x = expr]Inicjalizacja przechwyconej zmiennej (C++14)
[args...]Przechwytywanie packa parametrów (C++14)

5. Przykłady użycia

Przykład 1: Podstawowa lambda

auto print = []() { std::cout << "Hello Lambda!"; };
print(); // Wywołanie

Przykład 2: Lambda z parametrami

auto add = [](int a, int b) { return a + b; };
std::cout << add(5, 3); // 8

Przykład 3: Przechwytywanie zmiennych

int x = 10, y = 20;
auto lambda = [x, &y]() {
    std::cout << x << ", " << y;
    y++; // OK - przez referencję
    // x++; // Błąd - przez wartość (chyba że mutable)
};
lambda();

Przykład 4: Lambda w algorytmach STL

std::vector<int> v = {5, 3, 8, 1};
std::sort(v.begin(), v.end(), [](int a, int b) {
    return a > b; // Sortowanie malejące
});

Przykład 5: Lambda z typem zwracanym

auto div = [](int a, int b) -> double {
    if (b == 0) return 0.0;
    return static_cast<double>(a) / b;
};

Przykład 6: mutable

int counter = 0;
auto inc = [counter]() mutable {
    counter++;
    return counter;
};
inc(); // 1
inc(); // 2
// counter w zewnętrznym scope pozostaje 0

6. Zaawansowane zastosowania

Generic Lambdas (C++14)

auto generic = [](auto x, auto y) { return x + y; };
std::cout << generic(5, 3);   // 8
std::cout << generic(2.5, 1.5); // 4.0

Init Capture (C++14)

auto ptr = std::make_unique<int>(42);
auto lambda = [p = std::move(ptr)]() {
    std::cout << *p;
};

Immediately Invoked Lambda (IILE)

const auto result = [](int x) {
    // złożone obliczenia
    return x * x;
}(5); // natychmiastowe wywołanie

Rekurencyjne lambdy

auto factorial = [](int n) {
    auto f = [](auto&& self, int n) -> int {
        return n <= 1 ? 1 : n * self(self, n-1);
    };
    return f(f, n);
};
std::cout << factorial(5); // 120

7. Wydajność i optymalizacje

  • Lambdy bez przechwytywania mogą być konwertowane na wskaźniki funkcyjne
  • Lambdy z przechwytywaniem są obiektami funkcyjnymi (funktorami)
  • Inlineowanie - kompilator często inline’uje lambdy
  • Rozmiar zależy od liczby przechwyconych zmiennych
  • Wydajność zazwyczaj lepsza niż std::function

8. Różnice między wersjami C++

C++ WersjaNowe możliwości lambd

C++14Generic lambdas, init capture, mutable w const
C++17constexpr lambdy, capture *this
C++20Lambdy w unevaluated contexts, template lambdas

9. Najlepsze praktyki

  1. Unikaj domyślnego przechwytywania ([=], [&]) - bądź explicite
  1. Używaj init capture dla move-only typów (C++14)
  1. Preferuj lambdy nad std::bind
  1. Dla callbacków rozważ std::function gdy potrzebujesz type erasure
  1. Zachowaj prostotę - złożone lambdy utrudniają czytanie kodu
  1. Używaj constexpr lambd gdy możliwe (C++17)
  1. Unikaj mutable jeśli to możliwe - utrudnia zrozumienie kodu

Wyrażenia lambda to potężne narzędzie, które znacząco poprawia ekspresywność i elastyczność kodu w C++. Od C++11 stały się nieodłącznym elementem nowoczesnego programowania w tym języku.
