1. Słowo kluczowe auto

Co to jest?

auto to mechanizm dedukcji typu wprowadzony w C++11, który pozwala kompilatorowi automatycznie określić typ zmiennej na podstawie jej inicjalizatora.

Dlaczego używamy auto?

✔ Czytelność kodu - skraca zapis, szczególnie przy skomplikowanych typach

✔ Elastyczność - ułatwia refaktoryzację (zmiana typu inicjalizatora nie wymaga zmiany deklaracji)

✔ Unikanie niejawnej konwersji - typ jest dokładnie taki jak inicjalizator

✔ Wymuszenie inicjalizacji - zmienne z auto muszą być od razu zainicjalizowane

Składnia i przykłady

auto x = 42;            // int
auto y = 3.14;          // double
auto z = "Hello";       // const char*
auto const& ref = x;    // const int&

// Złożone typy
std::vector<std::string> names;
auto it = names.begin(); // std::vector<std::string>::iterator

// Szablony
template<typename T, typename U>
auto add(T a, U b) -> decltype(a + b) {
    return a + b;
}

Kiedy unikać auto?

❌ Gdy typ musi być dokładnie określony (np. interfejsy API)

❌ Przy inicjalizacji {} może dać nieoczekiwane typy (auto x{5} to std::initializer_list<int>)

2. Pętla for-each (range-based for loop)

Co to jest?

Pętla wprowadzona w C++11 do iteracji po elementach kontenera lub zakresu.

Dlaczego używamy for-each?

✔ Prostota - bardziej zwięzły zapis niż tradycyjne pętle

✔ Bezpieczeństwo - mniej miejsca na błędy off-by-one

✔ Uniwersalność - działa z wszystkimi kontenerami STL i typami z begin()/end()

Składnia

for(deklaracja : zakres) {
    // ciało pętli
}

Przykłady użycia

std::vector<int> numbers{1, 2, 3};

// Kopiowanie elementów (niezalecane)
for(int n : numbers) {
    std::cout << n << " ";
}

// Referencja (modifikowalna)
for(auto& n : numbers) {
    n *= 2;
}

// Stała referencja (tylko do odczytu)
for(auto const& n : numbers) {
    std::cout << n << " ";
}

// Wsparcie dla initializer_list
for(auto c : {'a', 'b', 'c'}) {
    std::cout << c;
}

Jak to działa?

Pętla jest tłumaczona na:

auto&& __range = zakres;
auto __begin = begin(__range);
auto __end = end(__range);
for(; __begin != __end; ++__begin) {
    deklaracja = *__begin;
    // ciało pętli
}

Własne typy z for-each

Aby użyć pętli z własnym typem, zaimplementuj:

class MyRange {
public:
    int* begin() { return data; }
    int* end() { return data + size; }
private:
    int data[10];
    int size = 10;
};

Połączenie auto i for-each

Najczęstszy idiom:

std::vector<std::string> words{"auto", "for", "each"};

for(auto const& word : words) {
    std::cout << word << " ";
}

Zalecane praktyki

  1. Zawsze używaj auto w for-each - zapewnia najlepszą wydajność
  1. Wybierz odpowiedni sposób przekazania elementu:
    • auto - kopiowanie (tylko dla małych typów)
    • auto& - modyfikowalna referencja
    • auto const& - tylko do odczytu
    • auto&& - uniwersalna referencja (perfect forwarding)

Podsumowanie - kiedy co stosować

KonstrukcjaNajlepsze zastosowanieUwagi
autoPrzy skomplikowanych typach, iteratorach, lambdachUnikaj z {} dla pojedynczych wartości

Złote zasady:
1.
auto + for-each to standard nowoczesnego C++
2.
Preferuj const& w pętlach jeśli nie modyfikujesz elementów
3.
Unikaj niepotrzebnych kopii - używaj referencji dla dużych obiektów
