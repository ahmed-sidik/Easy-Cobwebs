4. Template'y

1. Co to są template’y (szablony) w C++?

Szablony to mechanizm metaprogramowania w C++ pozwalający na pisanie kodu generycznego, który może działać z różnymi typami danych bez konieczności ich precyzowania podczas pisania kodu. Szablony są podstawą programowania generycznego w C++.

2. Po co wykorzystujemy template’y?

  • Unikanie duplikacji kodu: Piszesz algorytm raz, a działa dla wielu typów
  • Type safety: Lepsze niż makra czy void*, bo typy są sprawdzane w kompilacji
  • Wydajność: Kod jest generowany w kompilacji, brak narzutu w runtime
  • Biblioteki standardowe: STL (vector, list, sort itd.) używa szablonów
  • Metaprogramowanie: Obliczenia w czasie kompilacji (np. factorial<5>::value)

3. Składnia szablonów

Deklaracja szablonu funkcji:

template <typename T>
T nazwaFunkcji(T parametr) {
		// implementacja
}

Deklaracja szablonu klasy:

template <typename T, typename U = int> // U ma domyślny typ int
class NazwaKlasy {
    // ciało klasy
};

Szablony zmiennych (C++14):

template<typename T>
constexpr T pi = T(3.1415926535897932385);

Szablony z wieloma parametrami:

template <typename T, int size>
class Array {
    T arr[size];
    // ...
};

4. Przykłady użycia

Przykład 1: Prosta funkcja szablonowa

template <typename T>
T max(T a, T b) {
    return (a > b) ? a : b;
}

// Użycie:
int main() {
    std::cout << max<int>(5, 3);       // 5
    std::cout << max<double>(2.1, 3.7); // 3.7
    std::cout << max(5, 3);            // dedukcja typu - też działa
}

Przykład 2: Klasa szablonowa (kontener)

template <typename T, int N>
class Stack {
    T elements[N];
    int top = 0;
public:
    void push(T elem) { elements[top++] = elem; }
    T pop() { return elements[--top]; }
};

// Użycie:
int main() {
    Stack<std::string, 10> stringStack;
    stringStack.push("test");
    std::cout << stringStack.pop();
}

Przykład 3: Metaprogramowanie - obliczenia w kompilacji

template<int N>
struct Factorial {
    static const int value = N * Factorial<N-1>::value;
};

template<>
struct Factorial<0> {
    static const int value = 1;
};

// Użycie:
int main() {
    std::cout << Factorial<5>::value; // 120 (obliczone w kompilacji!)
}

Przykład 4: SFINAE (Substitution Failure Is Not An Error)

template<typename T>
typename std::enable_if<std::is_integral<T>::value, bool>::type
is_odd(T i) { return bool(i % 2); }

// Użycie:
int main() {
    is_odd(5);   // OK
    // is_odd(5.0); // Błąd kompilacji - tylko typy całkowite
}

Przykład 5: Variadic templates (C++11)

template<typename... Args>
void print(Args... args) {
    (std::cout << ... << args) << '\n'; // fold expression (C++17)
}

// Użycie:
int main() {
    print(1, 2.5, "hello"); // wypisze "12.5hello"
}

Szablony to potężne narzędzie, ale wymagają dobrego zrozumienia. W bardziej zaawansowanych zastosowaniach można spotkać się z:
- specjalizacją szablonów (pełną i częściową)
- SFINAE
- CRTP (Curiously Recurring Template Pattern)
- konceptami (C++20)
- szablonami wyrażeń (expression templates)
