Kontenery STL w C++

1. std::vector

Co to jest?

Dynamiczna tablica o zmiennym rozmiarze, zapewniająca szybki dostęp swobodny.

Charakterystyka:

  • Dostęp: O(1) przez operator []
  • Wstawianie/usuwanie na końcu: O(1) (amortyzowane)
  • Wstawianie/usuwanie w środku: O(n)
  • Realokacja: Powoduje invalidację wskaźników

Składnia:

std::vector<int> v = {1, 2, 3};
v.push_back(4);  // Dodanie elementu
int x = v[2];    // Dostęp

Zastosowania:

  • Główny kontener do przechowywania danych
  • Gdy potrzebny szybki dostęp indeksowany
  • Przy częstym dodawaniu tylko na końcu

2. std::map

Co to jest?

Asocjacyjny kontener przechowujący pary klucz-wartość w formie posortowanego drzewa (zazwyczaj RB-tree).

Charakterystyka:

  • Złożoność operacji: O(log n)
  • Elementy posortowane według klucza
  • Klucze unikalne

Składnia:

#include <map>
std::map<std::string, int> m;
m["apple"] = 5;     // Wstawianie
int x = m["apple"]; // Dostęp

Zastosowania:

  • Słowniki z wymogiem sortowania
  • Gdy potrzebny porządek leksykograficzny
  • Przy częstym wyszukiwaniu po kluczu

3. std::unordered_map

Co to jest?

Asocjacyjny kontener z haszowaniem, przechowujący pary klucz-wartość.

Charakterystyka:

  • Złożoność operacji: O(1) średnio, O(n) najgorszy przypadek
  • Elementy nieposortowane
  • Klucze unikalne

Składnia:

#include <unordered_map>
std::unordered_map<std::string, int> um;
um["banana"] = 3;   // Wstawianie
int x = um["banana"]; // Dostęp

Zastosowania:

  • Bardzo szybkie wyszukiwanie
  • Gdy sortowanie nie jest potrzebne
  • Słowniki z własnym hashem

4. std::queue

Co to jest?

Kolejka FIFO (First-In-First-Out).

Charakterystyka:

  • Dostęp tylko do początku i końca
  • Brak dostępu swobodnego
  • Zwykle implementowana na deque

Składnia:

#include <queue>
std::queue<int> q;
q.push(1);    // Dodanie
int x = q.front(); // Podgląd pierwszego
q.pop();      // Usunięcie pierwszego

Zastosowania:

  • Kolejki zadań
  • Przetwarzanie w kolejności przyjścia
  • Symulacje zdarzeń

5. std::stack

Co to jest?

Stos LIFO (Last-In-First-Out).

Charakterystyka:

  • Dostęp tylko do wierzchołka
  • Brak dostępu swobodnego
  • Zwykle implementowany na vectorze lub deque

Składnia:

#include <stack>
std::stack<int> s;
s.push(1);    // Dodanie
int x = s.top(); // Podgląd wierzchołka
s.pop();      // Usunięcie

Zastosowania:

  • Algorytmy stosowe (np. DFS)
  • Odwracanie kolejności
  • Kalkulatory RPN

6. Kopiec (std::priority_queue)

Co to jest?

Kontener implementujący kopiec (domyślnie max-heap).

Charakterystyka:

  • Dostęp do największego elementu: O(1)
  • Wstawianie/usuwanie: O(log n)
  • Elementy częściowo uporządkowane

Składnia:

#include <queue>
std::priority_queue<int> pq;
pq.push(3);     // Wstawianie
int x = pq.top(); // Podgląd maksimum
pq.pop();       // Usunięcie maksimum

Zastosowania:

  • Kolejki priorytetowe
  • Algorytm Dijkstry
  • Zarządzanie zadaniami z różnymi priorytetami

Wybór kontenera - praktyczne wskazówki

  1. Domyślny wybór: vector (chyba że potrzebujesz specjalnych właściwości)
  1. Potrzebujesz słownika?
    • Jeśli sortowanie ważne → map
    • Jeśli tylko szybkie wyszukiwanie → unordered_map
  1. Kolejkowanie zadań:
    • FIFO → queue
    • Priorytety → priority_queue
  1. Odwracanie kolejności/śledzenie historii: stack

Każdy kontener ma swoje optymalne zastosowania - kluczowe jest zrozumienie ich charakterystyki wydajnościowej! 🚀
